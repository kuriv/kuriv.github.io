错误控制运算符

PHP 支持一个错误控制运算符 @ 。当将其放置在一个 PHP 表达式之前，该表达式可能产生的任何错误信息都被忽略掉。目前的错误控制运算符 @ 甚至使导致脚本终止的严重错误的错误报告也失效。这意味着如果在某个不存在或者敲错了字母的函数调用前用了 @ 来抑制错误信息，那脚本会没有任何迹象显示原因而死在那里。

<?php

@require __DIR__ . '/example.php';
